## Что за продукт

Коммерческий SaaS-сервис для работы с договорами через ИИ.

Пользователи могут:
- Создавать договоры, приложения, допсоглашения с нуля через ИИ
- Загружать готовые документы и проверять их на риски
- Сравнивать версии документов
- Редактировать через ИИ-чат (встроенного ручного редактора нет)
- Хранить контрагентов с реквизитами и подписантами
- Оплачивать конкретные версии через внутренний баланс
- Скачивать уже купленные версии повторно бесплатно

**Целевая аудитория:** ИП, подрядчики, малые компании — те кто работает с договорами без юриста.

## База данных — главные сущности

```
User → Wallet (баланс)
User → Profiles (реквизиты: ИП / ООО / физлицо)
User → Counterparties (контрагенты)

Counterparty → BankDetails
Counterparty → Signatories (подписанты)
Counterparty → Documents

Document → Versions  ← APPEND-ONLY! Версии никогда не перезаписываются
Version → AISettings (уровень защиты, объём, инструкция)
Version → ChatMessages (история ИИ-чата этой версии)
Version → Purchase (куплена или нет)

Wallet → Transactions (история операций)
```

## Правила которые нельзя нарушать

1. **Версии — только добавлять.** Никогда не обновлять текст/файл существующей версии. Каждое изменение = новая версия.
2. **Файлы только в S3.** В базе данных хранить только путь к файлу, не сам файл.
3. **ИИ-запросы — в очередь.** Генерация документа через ИИ всегда идёт через BullMQ воркер, не в HTTP запросе напрямую.
4. **Покупка версии — транзакция БД.** Проверка баланса и списание должны быть атомарными (SELECT FOR UPDATE).
5. **ИИ-чат — постоянная колонка справа.** На рабочем экране чат не открывается поверх документа, он всегда рядом.
6. **Одна primary-кнопка на экран** (чёрная, главное действие).
7. **Интерфейс только на русском языке.**

## Что НЕ делаем в MVP

- ❌ Встроенный ручной редактор документа (только ИИ-чат)
- ❌ DaData / автозаполнение из ЕГРЮЛ (реквизиты вводятся вручную)
- ❌ OAuth через Госуслуги / Яндекс ID (но в схеме БД место для этого есть)
- ❌ Реальный платёжный шлюз (только внутренний баланс)
- ❌ Тёмная тема
- ❌ Командная работа / роли

## Как проверить что всё работает (e2e сценарий)

```
1. Зарегистрироваться / войти
2. Заполнить свои реквизиты (ИНН, банк и т.д.)
3. Создать контрагента, добавить его реквизиты и подписанта
4. Создать новый договор → выбрать контрагента → настроить ИИ
5. Открыть рабочий экран → увидеть сгенерированный документ
6. Через ИИ-чат попросить внести правку → создаётся новая версия
7. Проверить документ → увидеть оценку и замечания
8. Сравнить версию 1 и версию 2
9. Утвердить версию → баланс списывается
10. Скачать купленную версию → повторно скачать бесплатно
11. Проверить историю платежей и хранилище
```
